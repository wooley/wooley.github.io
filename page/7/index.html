康康的围棋之路

发表于 2012-12-24 | 分类于 成长路上

今年年初，康康4周岁多点。我那时会经常带他去栖霞棋院看大哥哥大姐姐们下棋。他在门口居然会认真的听老师上课。有一次，下课了，和他在教室摆摆棋，许老师还给他指点了几招呢。

二月份的时候，棋院新开了入门班，康康虽然还不足4周岁半，但看他还坐的住，老师还是让他报名了。刚好就是许斐老师带他们这个班，许老师上课节奏感很好，生动有趣，教室纪律也管理的很棒，小家伙们进步非常快，没过几次课，一个个都下的有模有样了。

时间过得很快，入门班、初级班，几个月就过去了。到了暑假，康康还参加了棋院的暑期集训班，这个暑假发生了很多事，爷爷做了2次手术，外公又意外去世，我们全家都乱了套。康康只能全天都托在棋院里，真是感谢棋院的老师们无微不至的照顾，梁老师，王老师，许老师等都很喜欢他呢。这次集训，进步非常快，他已经可以下完整盘棋，包括做活、对杀、连接已经都有点摸到门道了。

集训结束后，小顾参加了棋院举办定级赛，当然，这次只是重在参与而已。他的第一个对手是提高班的边进小朋友，小顾出来的时候表情，我知道他输了。不过，稍微安慰一下，他的心情就好多了，第二盘、第三盘，一胜一负。每盘都下的不轻松，不过他最大的乐趣就是去看分数牌。下午又大战了4盘，一胜三负。小顾第一次经历这个场面，看他比赛，我都比他紧张。这个成绩大概评到4级，虽然升段要有1级2级才有用，不过对他来说，这也是个不错的成绩了，王浩老师为了鼓励小顾，让我交了照片。等到小顾拿到评定的3级的证书时，开心极了，围棋兴趣大幅提高，看来鼓励真的很重要呢。

 

暑假结束，新的学期开始了，小顾升入了中班，围棋也进入中级班阶段。这段时间的死活题，已经有点难度了，有些题，我和康康会在棋盘上试验好久才能找到答案，唉，做陪练也不容易啊。

棋院每周六上午有免费提供的下棋指导，可以下完整盘棋后，坚持让他去多和各种对手下棋。不过经常去的一般都是提高班，高级班，段位班的小朋友，能和康康下的对手不多。记得第一去的时候，他还不太愿意，王老师他说下了就能有礼物，他才愿意下了一盘。第二次去，他还是不太愿意，王老师说下赢了才有礼物，小家伙居然讨价还价，问：“那下输了也给吧！”大家都被他逗乐了，不过那盘真给他下赢了。现在小顾是常客了，经常能下到3盘左右，这里真的很锻炼人，不同的对手，尤其大部分都比他厉害。他在实战中提高很快，从刚开是能吃掉人点棋就算不错到现在还经常能赢几局。

这个周日，围棋班进行了一次测验，他不但第一个完成，而且全部答对。许老师为了鼓励小朋友们，等考试结束后，颁发了奖励。康康第一个领奖，是他最近一直眼馋的一个战斗陀螺。

学围棋让他感觉到了快乐，这样才能真正有学习的动力，感觉棒极了。

寒假即将到来，希望这次的升级赛，能获得更好的成绩。加油，康康。

于是，按照原来的方法，找个最便宜的服务，现在是 日历服务 http://www.securepaynet.net/calendar/online-calendar.aspx?ci=1798&prog_id=485480 每年费用是4.99$.加上.me域名续费，嘿嘿，续费费用直接变为了1.99，合起来7美元不到，就可以续费一年了。包括买新的域名，同样可以这样操作，不过新域名有个1美元多的税，合起来也是相当便宜。所以，我又买了个 qxqy.net 的域名，儿子在栖霞棋院学棋，就当是个礼物送给棋院。
还不借这个机会赶紧把你的域名多续费几年。快下手吧。

暗黑3v1.04巅峰等级快速冲级攻略z

发表于 2012-12-11 | 分类于 生活点滴

A1
对于装备不是很好，或者纯防御装的童鞋，流畅刷A1要比拖尸体刷A2来的效率的多。
所以推荐DPS1W左右的骚年们来这里升级，下面是升级路线
选“被囚禁的天使”任务，记得是191吧
——进入新崔斯特姆
——先传送腐溃之林，这里必刷2-3组精英，然后两个洞里的精英，直接4-5BUFF回城，注意这里还是不要清小怪，一则小怪少又不集中，而且那幽灵打起来比较费劲，再则没有5BUFF前刷怪的行为时不明智的
——回城后传北高地，沿路小怪真的是成群结队，推荐不要放过一群小怪，一两只的话倒是无所谓，特别提醒，传送点左面有50%几率刷瞭望塔，怪并不多，但是掉非常多的金币，里面必刷1-2组精英和1个华丽包厢，速刷经验可以无视，但是对那些修不起装备的屌丝们，还是去拿点福利吧(^o^)/~
——一路蛇形向上推进，来到李奥瑞克大院，门口必刷1-2组精英，也可以跳过直接进屋刷大厅的31只又肥又嫩的小怪
——后门出来后进刑牢1层，一路没什么可说的，只是这里容易迷路啊，走到胡同里的心情真是郁闷啊o(╯□╰)o
——顺图一直跑到最后，屠夫不建议刷，没什么意义，有点浪费时间，但是考虑到屠夫后门有20来个小怪，顺便打一下也不是不行了啦!
A2
刷不了A3的童鞋混A2其实也是个好主意，但就刷经验而言A2一点不比A3慢，真的是一点不慢
但是能刷A3还是去A3吧，毕竟掉落没法比
选任务记得选黑灵魂石—273任务
——第一趟刷先传凄凉沙地，第一时间找到刺客地库的入口，顺路杀杀不知死活的拉库尼，秃鹫虾米的，但是别忘了第一目标是地库，也不是背叛者洞窖
——进入地库后有一个关键的存储点OK，这就是我们后面的出发地了!地库相信多数盆友已经轻车熟路了，一路上至少7组精英，最多3只哥布林，唯一的问题是金色火属性长脚了，打起来很窝火，还容易挂，大家小心点。地库刷下来你就是绕着精英走都能刷到5层BUFF，下面正式开刷!
——黑谷矿坑，传送点旁边就有3,4队的沉沦魔，还有可能刷最多3组精英，刷之。再往左走穿过小门，忘记地名了。这个谷底里，4,5组小怪而且地图中间必刷1个紫名怪，紫名怪的经验可是超多的!刷完这里再次回城
——这里可以穿奥卡纳斯，也可以去卡辛姆哨站，其实效率差不多的，直接往奥卡纳斯里面走，一路打打杀杀的，好无聊，清到马大姐门前，OK，打道回府咯!
——然后可以退出重置，也可以去绿洲逛逛，运气好怪多，运气不好……杀个紫名，然后重置
值得一提的是重置之后再进出现在地库的存档点，这样上线直接刷地库，直接5BUFF!效率就体现在这了!但是千万别进灵魂石大厅，会重置存档点的
A3
总体来讲我个人喜欢刷A3，因为，A2真的很没感觉。所谓的木有激情啊!我的几件传奇装备都是在A3打出来的。不说好坏，我在A2刷到10级就木见过。
A3的有的组合精英对我这种装备不好的来说还是有点挑战的，所以说刺激，还是有滴!
昨天逛贴吧，看见居然有人在讨论通刷，就是从311开始做任务，我赶脚很无语……
下面大家参考一下我的A3路线
——372任务是可行的，如果打蜘蛛不能0上天，或者真的好烦这女……蜘蛛在耳边嗡嗡，可以选择373也行
——进游戏很多人都去刷坑?去要塞?其实正确的做法是先传塔1，这里小怪相对少，精英稍多，刷完塔1你肯定有5BUFF在手了，这时不要进坑2，回城收拾一下行李，进坑1刷可爱的小蝎子吧!如果进来就刷坑的话相当于这里的经验少了一半，而这里的怪，像蝎子，是非常多且肥的!
——正常刷完坑，这里推荐不放过每一个角落，因为一个角落里就可能藏着60多只蝎子……坑1然后再回程，坑2。后面就一直沿途清到摸蛋。这里说一下，如果选的372，就一路清，选的373的话诅咒2层清完直接回城传送地核，因为整个诅咒底层一只怪都没有
——刷完蛋蛋，回城传大桥，注意，不是传拉基斯路口，是大桥，传完大桥按下面的路线走：直接一直想做走，引到自己能一波请掉那么多怪时，停下清掉;转而向上走，这里最好沿着大路中间走，这样能最大限度引怪，就是别紧贴边走;走到看见右面有一个坑的时候，右转，从小路下去这样一直向下走，就能回到大桥传送点的右侧了，这里完全没有走冤枉路，这样一路向上S形前进把所有怪杀光，一直杀到路口;这是别着急进路口，由于路线选择关系路口左面一点的怪没有清到，这时勒马左转，杀个回马枪去也!!!
——然后可以直接进路口，如果包裹太沉可以回去清理一下，然后从传送点传到拉基斯路口，这里的效率是没有差别的。沿路上/下清至传送点/杀戮平原，结束这段路程。路口的怪真的很多，而且非常弱。这里刷3个任务，个人推荐除了队长的任务，都不予理睬，因为完全是浪费时间。
——最后可以选择是否刷要塞。城墙的效率极低，个人感觉，所以我没有刷过城墙。而要塞我本人也是不推荐，效率也不高，而且容易迷路。这里鄙视一下这里的设计师，总是选一条路走了10分钟发现最后走到了一件洗手间，不得不掉头往回走……
整个A3最效率的地点就是杀戮平原，就是从大桥传送点出来的地方。甚至可以把行程直接缩减成：赎罪塔+坑1+平原然后重置，看个人喜好了
以上是个人推荐的效率线路。A4,并没有效率，而且阵亡率偏高，只是掉落更容易弥补损失。
而下面是我的一些建议：
1. 两人刷，效率最高，尤其是法师+蛮子，DH也不错的。和尚，51还是适合单刷。
2. 随从方面，MM加15%护甲和3攻速，偏防御;林登的3爆让你刷的更快;寇马克是我最推荐的!很多人不解，但是就升级来讲，寇马克能穿6件装备!那就是6×6提供36点经验加成!在护甲不缺的情况下还是带寇马克吧，给他配上+24经验+MF的一身，绝对然你事半功倍!
3. 真的不推荐通刷，很没效率，因为任务并不奖励经验值，纯属耽误时间!
4. 头盔的问题大家可能都知道的，但是我还是提一下，以25%的为例，如果一趟刷1000W经验，5BUFF给你加750W经验，宝石给你加250W，所以没宝石是1750W，而镶上后是2000W，所以石头的实际效果是13.5%经验，希望大家了解。
5. 路上绝对要克制自己，不要捡蓝装，一件蓝装只有300-1800G，你要告诉自己，“这是在那G换经验值!paragon100以后这一地的全都要变成暗金色的!不会掉好东西，不会掉好东西……”黄装也只捡61，62，63。

ODB在1.8.0版本开始支持mssql

发表于 2012-12-11 | 分类于 程序人生

Support for the Microsoft SQL Server database. The provided connection
factories include ‘new’ (a new connection is created every time one is
requested) and ‘pool’ (a pool of connections is maintained). The Boost
and Qt profiles have been updated to support this database. For more
information, refer to Chapter 17, “Microsoft SQL Server Database” in
the ODB manual.

Note that with the release of SQL Server Native Client ODBC driver for
Linux it is possible to access an SQL Server database from the GNU/Linux
operating systems. The SQL Server ODB runtime has been tested and is
fully supported on both Windows and GNU/Linux. However, we have found
that the unixODBC 2.3.0 driver manager that is recommended in the Native
Client documentation does not work well, especially in multithreaded
applications. The recently released unixODBC 2.3.1 addressed this issue
and we recommend that you use this version with ODB. For more information,
refer to this blog post:

http://www.codesynthesis.com/~boris/blog/2011/12/02/microsoft-sql-server-odbc-driver-linux/

Note also that similar to Oracle, the SQL Server runtime (libodb-mssql)
is licensed under a different license compared to the open source
database runtimes. For more information, refer to the ODB License page:

http://www.codesynthesis.com/products/odb/license.xhtml

netstat的全部参数及说明

发表于 2012-12-11 | 分类于 程序人生

备注一下，容易忘。

显示协议统计信息和当前 TCP/IP 网络连接。
NETSTAT [-a] [-b] [-e] [-n] [-o] [-p proto] [-r] [-s] [-v] [interval]
-a 显示所有连接和监听端口。
-b 显示包含于创建每个连接或监听端口的
可执行组件。在某些情况下已知可执行组件
拥有多个独立组件，并且在这些情况下
包含于创建连接或监听端口的组件序列
被显示。这种情况下，可执行组件名
在底部的 [] 中，顶部是其调用的组件，
等等，直到 TCP/IP 部分。注意此选项
可能需要很长时间，如果没有足够权限
可能失败。
-e 显示以太网统计信息。此选项可以与 -s
选项组合使用。
-n 以数字形式显示地址和端口号。
-o 显示与每个连接相关的所属进程 ID。
-p proto 显示 proto 指定的协议的连接；proto 可以是
下列协议之一: TCP、UDP、TCPv6 或 UDPv6。
如果与 -s 选项一起使用以显示按协议统计信息，proto 可以是下列协议
之一:
IP、IPv6、ICMP、ICMPv6、TCP、TCPv6、UDP 或 UDPv6。
-r 显示路由表。
-s 显示按协议统计信息。默认地，显示 IP、
IPv6、ICMP、ICMPv6、TCP、TCPv6、UDP 和 UDPv6 的统计信息；
-p 选项用于指定默认情况的子集。
-v 与 -b 选项一起使用时将显示包含于
为所有可执行组件创建连接或监听端口的
组件。
interval 重新显示选定统计信息，每次显示之间
暂停时间间隔(以秒计)。按 CTRL+C 停止重新
显示统计信息。如果省略，netstat 显示当前
配置信息(只显示一次)

修复方式：

在include目录下的system.func.php 里，把

$pdata[‘motto’] =$db->result($result2, 0);

$pdata[‘motto’] = str_replace(“‘“, “‘’”, $db->result($result2, 0));
